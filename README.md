# GP_2_GameSphere_Analysis

## Описание проекта

Исследование посвящено анализу рынка компьютерных и видеоигр на основе нескольких источников данных:

- сайт **Steam**
- сайт **RAWG**
- данные **YouTube**

Цель работы - получить прикладные инсайты для индустрии игр: понять, как связаны **цена, рейтинги, жанры, платформы и медиа-активность**, а также определить, какие типы игр показывают наилучшие метрики.

## Сбор данных: Steam (`Steam_scraper.ipynb`)

Данные собирались парсингом выдачи поисковой строки онлайн-магазина **Steam**.  
Для автоматизации браузера, перехода по страницам и извлечения информации об играх использовалась библиотека **Selenium**.

### Итоговый датасет

В итоговом датасете - около **20 000 строк**, где каждая строка соответствует отдельной игре.

### Признаки датасета

- `name` - название игры
- `description` - текстовое описание
- `price` - цена в Steam в текстовом виде (например, `37.99 USD`, `Free To Play`)
- `reviews` - словесная оценка (`Very Positive`, `Mostly Positive` и др.)
- `total_reviews` - общее количество отзывов
- `total_positive` - число положительных отзывов
- `total_negative` - число отрицательных отзывов
- `system_requirements` - системные требования
- `tags` - список тегов
- `url` - ссылка на страницу игры в Steam

### Дополнительно

- В ноутбуке настроено логирование процесса парсинга
- Параметры логирования можно изменять в `config.json`
- Логирование можно отключать при необходимости
- Логи сохраняются в файл `scraper.log`, который добавлен в `.gitignore`
- Собранные данные сохраняются в файл `steam_games.csv`


## Сбор данных: RAWG (`api_rawg.ipynb`)

В этом ноутбуке используется API сервиса **RAWG** для получения информации об играх.  
Было выполнено **3 запроса к API** для выборки популярных и релевантных игр.

### Итоговый датасет

В итоговом датасете около **20000 строк**.

### Признаки датасета

- `id` - идентификатор игры в RAWG
- `name` - название игры
- `released` - дата релиза
- `rating` - пользовательский рейтинг игры по шкале от 0 до 5
- `ratings_count` - количество оценок и отзывов
- `metacritic` - агрегированный критический рейтинг по шкале от 0 до 100
- `playtime` - среднее количество часов прохождения
- `genres` - жанры игры (например, `Strategy`, `Simulation`)
- `platforms` - платформы и устройства, на которых доступна игра
- `screenshots_count` - количество скриншотов на сайте RAWG
- `achievements_count` - количество достижений в игре

Итоговый CSV-файл сохраняется как `rawg_games20.csv`.

## Объединённый анализ игр  
### `visualization_games.ipynb`, `EDA_2-1.ipynb`

### `visualization_games.ipynb`

Проводится визуальный анализ двух игровых датасетов (**Steam** и **RAWG**):

- составление визуализаций для дальнейших гипотез

### `EDA_2-1.ipynb`

Выполнен подробный **EDA**:

- объединение датасетов Steam и RAWG по названию игры (получено примерно 5000 строк)
- анализ распределений и пропусков
- обработка текстовых и категориальных признаков
- изучение связей между отзывами, рейтингами и ценой
- предсказание переменной metacritics с помощью алгоритма **регрессии**, для получения весов модели и более глубокого понимания влияния факторов на важный признак, такой как оценка критиков

## Инсайты и исследовательская задача

Основная исследовательская задача — понять, какие характеристики игр связаны с **популярностью** и **коммерческим успехом**, чем отличаются хиты от нишевых проектов.

EDA даёт практические инсайты для сферы компьютерных игр:

- какие жанры растут
- какие платформы чаще встречаются в топ-играх
- как цена соотносится с рейтингами и отзывами
- какие паттерны можно учитывать при выпуске новых проектов

## Совместный анализ игр и YouTube

В отдельном ноутбуке выполняется объединение датасета игр с данными по YouTube-видео.

В рамках этого анализа:

- изучили влияют ли показатели популярности ролика (просмотры, лайки, комментарии) на рейтинг игры
- посмотрели зависимости между метриками игры и показателем вовлечение на роликах
- сделали вывод о релевантности маркетингового продвижения на платформе
